### 1. Linux 기초

서버는 터미널로 관리해요. 아래 명령어들은 앞으로 매주 사용하게 될 겁니다.
모두 외우려고 노력하지 않아도 됩니다. 어차피 쓰다보면 익을거에요. 옆에 띄워두고 필요할 때 따라 치거나, 아니면 따로 인터넷 검색으로 찾아도 좋아요.

#### 파일 시스템 탐색

```bash
pwd                    # 현재 디렉토리 (Print Working Directory)
ls -la                 # 파일 목록 (숨김 파일 포함, 상세 정보)
cd /home/username      # 디렉토리 이동
cd ..                  # 상위 디렉토리
cd ~                   # 홈 디렉토리
cat file.txt           # 파일 내용 출력
less file.txt          # 파일 내용 스크롤 (q로 종료)
head -n 20 file.txt    # 처음 20줄
tail -f logfile.log    # 마지막 줄부터 실시간 추적 (로그 볼 때 유용!)
```

#### 파일/디렉토리 조작

```bash
mkdir -p project/src   # 디렉토리 생성 (-p: 중간 경로도 생성)
cp file.txt backup/    # 복사
mv old.txt new.txt     # 이름 변경 / 이동
rm file.txt            # 삭제 (되돌릴 수 없어요!)
rm -rf directory/      # 디렉토리 통째로 삭제 (조심!)
chmod 755 script.sh    # 권한 변경
chown user:group file  # 소유자 변경
```

#### 프로세스 관리

```bash
ps aux                 # 실행 중인 프로세스 목록
ps aux | grep node     # node 관련 프로세스만 필터
top                    # 실시간 시스템 모니터 (q로 종료)
htop                   # top의 개선 버전 (설치 필요)
kill <PID>             # 프로세스 종료
kill -9 <PID>          # 강제 종료
```

- `Ctrl+C`: 실행 중인 프로그램 종료
- `Ctrl+Z`: 백그라운드로 보내기 → `bg`로 계속 실행, `fg`로 다시 포어그라운드

#### 기본 네트워크 도구

```bash
curl http://example.com          # HTTP 요청 보내기
curl -v http://example.com       # 상세 출력 (헤더까지!)
ss -tlnp                         # 열려있는 포트 확인
ip addr                          # 네트워크 인터페이스 & IP 확인
```

#### 텍스트 편집

```bash
nano file.txt          # 간단한 편집기 (초보자 추천)
vi file.txt            # vim 편집기 (i: 편집, Esc: 명령, :wq: 저장 종료)
```

#### 패키지 관리

```bash
sudo apt update                  # 패키지 목록 갱신
sudo apt install <패키지명>       # 패키지 설치
sudo apt remove <패키지명>        # 패키지 삭제
```

### 2. HTTP 프로토콜

HTTP(HyperText Transfer Protocol)는 웹에서 데이터를 주고받는 가장 기본적인 프로토콜이에요. 우리가 만들 서버는 HTTP로 통신합니다.

#### 요청-응답 모델

HTTP는 **클라이언트가 요청(Request)을 보내면, 서버가 응답(Response)을 보내는** 단순한 구조예요. 그리고 **무상태(Stateless)**입니다 — 서버는 그 자체로는 여러분이 보낸 이전 요청을 기억하지 않아요.

#### HTTP 요청의 구조

```
POST /api/users HTTP/1.1          ← 메서드, 경로, 버전
Host: localhost:3000               ← 헤더 (메타데이터)
Content-Type: application/json

{"name": "홍길동"}                  ← 본문 (Body, 선택)
```

#### HTTP 메서드

| 메서드 | 용도 | 예시 |
|--------|------|------|
| GET | 데이터 조회 | 사용자 목록 가져오기 |
| POST | 데이터 생성 | 새 사용자 등록 |
| PUT | 데이터 수정 | 사용자 정보 업데이트 |
| DELETE | 데이터 삭제 | 사용자 삭제 |

#### HTTP 상태 코드 — 꼭 알아야 하는 것들

| 코드 | 의미 | 언제 보게 되나 |
|------|------|------------|
| `200` | OK | 정상 응답 |
| `201` | Created | POST로 리소스 생성 성공 |
| `301/302` | Redirect | HTTP → HTTPS 리다이렉트 |
| `400` | Bad Request | 요청 형식이 잘못됨 |
| `401` | Unauthorized | 인증 안 됨 |
| `403` | Forbidden | 권한 없음 |
| `404` | Not Found | 경로가 없음 |
| `500` | Internal Server Error | 서버 코드에서 에러 발생 |
| `502` | Bad Gateway | 프록시 뒤의 서버가 응답 안 함 |
| `503` | Service Unavailable | 서버 과부하 / 점검 중 |

#### 헤더

- `Content-Type`: 본문의 형식 (예: `application/json`, `text/html`)
- `Host`: 요청 대상 서버
- `User-Agent`: 클라이언트 정보 (브라우저, curl 등)
- `Authorization`: 인증 토큰

### 3. JavaScript & Node.js 기초

JavaScript는 웹 브라우저용 언어로 시작했지만, **Node.js** 덕분에 서버에서도 실행할 수 있게 되었어요. Node.js는 서버에서 JavaScript를 읽고 실행하는 런타임입니다.

아래 문법은 이번 주 실습과 앞으로의 Express.js 개발에 필요한 최소한의 내용이에요.

- 변수 (let, const)
- 자료형 (Number, String, Boolean, Array, Object)
- 조건문 (if), 반복문 (for)
- 함수 (function, arrow function)
- **비동기 (Promise, async/await)** → 🔥 중요!
- Node.js 내장 모듈: `os`, `fs`, `http`

### 실습: OS 정보를 보여주는 HTTP 서버 만들기

Express 없이, `http` 모듈만으로 간단한 서버를 만들어봐요. 서버를 실행하고 `http://서버IP:3000`에 접속하면 OS 정보가 JSON으로 응답되어야 합니다.

#### 요구사항

1. `http` 모듈로 서버를 만드세요. 포트는 **3000번**으로 지정하세요.
2. Express를 쓰지 않고 `http` 모듈만으로 구현하세요. 파일 하나(`server.js`)면 충분해요!
3. 아래 두 개의 endpoint를 구현하세요.

#### `GET /`

JSON으로 OS 정보를 반환합니다. `os` 모듈을 사용하세요.

```json
{
  "type": "Linux",
  "hostname": "infra-study-01",
  "cpu_count": 4,
  "total_memory_mb": 8192,
  "uptime_hours": 123.45
}
```

#### `GET /health`

서버가 살아있는지 확인하는 헬스체크 endpoint입니다.

```json
{
  "status": "ok"
}
```

이 endpoint는 사소해 보이지만, **모든 프로덕션 서비스에 존재해요.** 나중에 Kubernetes의 liveness probe에서 이 endpoint를 사용하게 됩니다.

#### 그 외 경로

존재하지 않는 경로에 요청이 오면 `404` 상태 코드와 함께 에러 메시지를 반환하세요.

### 테스트

서버에서 실행 후, 다른 터미널 또는 로컬에서 `curl`로 테스트해보세요.

```bash
# 서버에서 실행
node server.js

# 다른 터미널에서 테스트
curl http://서버IP:3000/
curl http://서버IP:3000/health
curl http://서버IP:3000/nonexistent    # 404 확인

# -v 옵션으로 HTTP 헤더까지 확인해보세요!
curl -v http://서버IP:3000/
```
