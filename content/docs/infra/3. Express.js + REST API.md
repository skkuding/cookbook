지난 주에는 `http` 모듈로 직접 서버를 만들어봤어요. 라우팅, 헤더 파싱, 응답 처리를 전부 직접 해야 해서 불편했죠? **Express.js**는 이런 반복 작업을 대신 해주는 웹 프레임워크예요.

이번 주에는 Express.js로 제대로 된 REST API를 만들고, **PostgreSQL 데이터베이스**를 연동해볼 거예요. 이 앱이 Session 8까지 계속 발전하게 됩니다!

## 공부할 내용 📚

### 1. REST API

REST(Representational State Transfer)는 API를 설계하는 아키텍처 스타일이에요. HTTP 메서드와 URL 경로로 리소스에 대한 작업을 표현합니다.

| 작업 | HTTP 메서드 | 경로 | 예시 |
|------|------------|------|------|
| 목록 조회 | GET | `/api/todos` | 전체 할 일 조회 |
| 단건 조회 | GET | `/api/todos/:id` | 특정 할 일 조회 |
| 생성 | POST | `/api/todos` | 새 할 일 추가 |
| 수정 | PUT | `/api/todos/:id` | 할 일 수정 |
| 삭제 | DELETE | `/api/todos/:id` | 할 일 삭제 |

- URL은 **명사** (리소스)를 나타내고, HTTP 메서드가 **동사** (행위)를 나타내요.
- 응답은 JSON 형식으로 보내는 게 표준이에요.

### 2. Express.js

Express.js의 핵심 개념들을 공부해보세요.

- **라우팅:** `app.get()`, `app.post()`, `app.put()`, `app.delete()`
- **미들웨어:** `app.use()` — 요청이 핸들러에 도달하기 전에 거치는 함수. `express.json()`이 대표적 (JSON body 파싱)
- **라우트 파라미터:** `req.params.id`, **쿼리 파라미터:** `req.query.search`
- **응답:** `res.json()`, `res.status()`
- **오류 처리:** 에러 핸들링 미들웨어

#### 참고 자료

- **[생활코딩 Express.js 강의](https://opentutorials.org/course/3370)**: 조금 오래되었지만 인기 있는 강의입니다.

### 3. PostgreSQL 기초

지금까지는 데이터를 메모리(배열)나 JSON 파일에 저장했어요. 서버가 재시작되면 데이터가 사라지죠. **데이터베이스**를 사용하면 데이터가 영구적으로 보존됩니다.

PostgreSQL은 가장 많이 쓰이는 오픈소스 관계형 데이터베이스예요.

- **기본 SQL:** `CREATE TABLE`, `INSERT`, `SELECT`, `UPDATE`, `DELETE`
- **데이터 타입:** `SERIAL` (자동 증가 ID), `VARCHAR`, `BOOLEAN`, `TIMESTAMP`
- **연결:** Node.js에서는 `pg` 패키지로 연동

이번 주에는 **실습 서버에 PostgreSQL을 직접 설치해서** 사용해요. Session 4에서 이걸 Docker 컨테이너로 대체하고, Session 5에서 AWS RDS로 이관할 거예요.

#### 참고 자료

- **[생활코딩 "SQL" (강의)](https://opentutorials.org/course/3161)**: SQL 기초를 배울 수 있는 강의입니다.
- **[PostgreSQL Tutorial](https://www.postgresqltutorial.com/)**: 영어지만 매우 체계적인 튜토리얼입니다.
- **[node-postgres 공식 문서](https://node-postgres.com/)**: Node.js에서 PostgreSQL을 사용하는 `pg` 패키지 문서입니다.

### 4. 환경 변수

데이터베이스 접속 정보 같은 설정값은 코드에 하드코딩하면 안 돼요. **환경 변수**로 분리합니다.

```bash
# 환경 변수 설정 후 실행
DATABASE_URL=postgres://user:pass@localhost:5432/tododb PORT=3000 node src/index.js
```

```javascript
// 코드에서 사용
const port = process.env.PORT || 3000;
const dbUrl = process.env.DATABASE_URL;
```

이 패턴은 Docker(환경 변수 주입), Kubernetes(ConfigMap/Secret), AWS(Parameter Store) 전부에서 동일하게 사용돼요.

## 프로젝트 실습 🎈

### 환경 세팅

```bash
# 프로젝트 초기화
mkdir todo-app && cd todo-app
npm init -y

# package.json의 type을 module로 설정 (ES6 import 사용)
# "type": "module" 추가

# 패키지 설치
npm install express pg
npm install --save-dev nodemon    # 코드 변경 시 자동 재시작
```

`package.json`에 스크립트를 추가하세요:

```json
{
  "type": "module",
  "scripts": {
    "dev": "nodemon src/index.js",
    "start": "node src/index.js"
  }
}
```

### PostgreSQL 설치 및 설정 (실습 서버에서)

```bash
# PostgreSQL 설치
sudo apt update
sudo apt install -y postgresql

# PostgreSQL 서비스 시작
sudo systemctl start postgresql
sudo systemctl enable postgresql

# 데이터베이스와 사용자 생성
sudo -u postgres psql
```

```sql
-- psql 안에서 실행
CREATE USER todouser WITH PASSWORD 'todopass';
CREATE DATABASE tododb OWNER todouser;
\q
```

```bash
# 접속 테스트
psql -U todouser -d tododb -h localhost
# 비밀번호: todopass
```

### 구현할 API

#### `GET /health`

```json
{ "status": "ok" }
```

#### `GET /api/todos`

모든 할 일을 조회합니다.

```json
[
  {
    "id": 1,
    "title": "Docker 공부하기",
    "completed": false,
    "created_at": "2025-03-20T12:00:00Z"
  }
]
```

#### `POST /api/todos`

새 할 일을 추가합니다.

Request Body:
```json
{ "title": "Kubernetes 공부하기" }
```

Response (201 Created):
```json
{
  "id": 2,
  "title": "Kubernetes 공부하기",
  "completed": false,
  "created_at": "2025-03-20T13:00:00Z"
}
```

#### `PUT /api/todos/:id`

할 일을 수정합니다.

Request Body:
```json
{ "title": "Docker 마스터하기", "completed": true }
```

#### `DELETE /api/todos/:id`

할 일을 삭제합니다. (204 No Content)

### 데이터베이스 테이블

```sql
CREATE TABLE todos (
    id SERIAL PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    completed BOOLEAN DEFAULT false,
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 테스트

```bash
# 서버 실행
DATABASE_URL=postgres://todouser:todopass@localhost:5432/tododb npm run dev

# API 테스트
curl http://서버IP:3000/health

curl -X POST http://서버IP:3000/api/todos \
  -H "Content-Type: application/json" \
  -d '{"title": "Docker 공부하기"}'

curl http://서버IP:3000/api/todos

curl -X PUT http://서버IP:3000/api/todos/1 \
  -H "Content-Type: application/json" \
  -d '{"completed": true}'

curl -X DELETE http://서버IP:3000/api/todos/1
```
